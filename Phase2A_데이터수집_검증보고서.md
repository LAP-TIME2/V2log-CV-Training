| 300장 | 75~80% | **55~75%** | 과장 |
| 500장 | 80~85% | **65~80%** | 부분 과장 |
| 1,000장 | 85~88% | **75~85%** | 부분 과장 |

**근거**: YOLO 논문 실험에서 1,504장으로 mAP50 = 96% 달성한 사례도 있지만, 그건 단순한 물체(아프리카 야생동물 4종). 플레이트는 비슷하게 생긴 물체를 무게별로 구분해야 해서 더 어려움.

### 데이터 증강: 3배가 한계

| 주장 | 검증 결과 |
|------|----------|
| "100장 → 10,000장 증강 가능" | **기술적으로 가능하나 3배(3x)가 Roboflow 공식 권장 한도** |
| "증강이 실제 데이터를 대체" | **❌ 과장됨. 보충이지 대체가 아님** |
| "합성 80% + 실제 20% 혼합" | **✅ 검증됨. 논문에서 가성비 최적점으로 확인** |

### DALL-E/Midjourney가 안 되는 이유 (원본 문서의 오류)

원본에서 "DALL-E로 플레이트 이미지 생성 → 쉬움"이라 했는데 **실제로는 비효율적**:
1. 생성한 이미지에 **바운딩 박스 좌표가 없음** → 수동 라벨링 필요 (핵심 이점 상실)
2. 플레이트 위치/각도 **정밀 제어 불가**
3. 장당 $0.04~0.08 → 10,000장 = **$400~800** (비쌈)

**대안**: Stable Diffusion + ControlNet (무료, 로컬 실행)이 훨씬 나음. 하지만 이것도 라벨링 문제는 있음.

## Part 3 — 현실 로드맵

### 원본 문서에서 가장 중요했던 포인트: 검증됨 ✅

> **"처음부터 5,000장이 필요한 게 아니다. 200장으로 시작해서 점진적으로 늘리면 된다."**

이건 논문으로 **검증된 사실**. 실제 연구에서:
- 실제 데이터 50장 + 합성 데이터 = 실제 데이터 400장과 **동등한 성능**
- Transfer Learning으로 500장이면 **서비스 시작 가능 수준**

### 수정된 현실 로드맵 (16만원 이하)

```
[즉시] 0원으로 시작                              ← 이번 주
├── Roboflow 공개 데이터셋 다운로드 (~10,000장)
├── 무게별 라벨은 없지만 "플레이트 찾기" 기본 능력 학습
└── HackerFit 데이터셋의 YOLOv11 사전학습 모델 테스트

[토요일] 직접 촬영 200장                          ← 비용 0원
├── 본인 헬스장에서 무게별 10~15장씩
├── 이게 "무게별 라벨 있는" 유일한 데이터
└── Roboflow Label Assist로 반자동 라벨링

[1~2주] Transfer Learning MVP                     ← 비용 0원
├── 공개 데이터로 사전학습 → 200장으로 Fine-tuning
├── 데이터 증강 3x = 600장
├── 기대 정확도: mAP50 65~80%
└── "사진 찍으면 AI가 추측 → 원탭 확인" UX로 충분

[1~2개월] 저비용 확장                              ← 비용 5~16만원
├── 헬스장 2~3곳 파트너십 (0원)
├── 추가 500장 촬영 → 총 700장
├── 크라우드웍스 라벨링 외주 (6~11만원)
└── 기대 정확도: mAP50 75~85%

[3개월+] 자동 수집 체계                            ← 비용 0원
├── V2log 앱 내 "플레이트 등록" 기능
├── 사용자 수정 데이터가 자동으로 학습 데이터가 됨
└── 플라이휠: 사용자↑ → 데이터↑ → 정확도↑ → 사용자↑
```
